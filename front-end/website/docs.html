Overview

A Server requires qubits for computation. The world-wide atack only crippled the server computational power but the physical network is still functional. Most servers have a few qubits but need some help to become operational. Your organisation has physical access to a few servers around the world from where you will mount your invasive. All Servers are equipped with

  • an Isolator Machine that can stabilise and isolate the qubits and protect from decoherence.
  • a Generator Machine that can "create" qubits and the generation rate is proportional to the qubits on the machine [1].

The Isolator malfunctions if qubits from a different Server system come onto the Server from the network, and qubits are lost proprtional to the amount of attack qubits that arrive. When all host qubits are destroyed, Server control is lost and the attacker gains control.

Computational Power (Currency of the Game)

The number of qubits that reside on your machine is the machine's `power`. You own more qubits that are in the custody of the routers but they are not available for computation and hence do not contribute to `power`. Neutral Servers

There are neutral servers that are healhty but under nobody's command, as only those servers to which humans had physical access could be operational. But, with an operational server, it is possible to command these neutral servers and use their resources for a common goal.

The neutral servers are in a non-operational "submissive-state", where the isolator and generator are turned off but there are qubits available. If enough qubits arrive from a certain Server Cluster, this server will join the cluster (isolator will become operational) and all the resident qubits will become available. This is a very cheap way of getting qubits as not only do you get a qubit source, but also a bonus amount of qubits!

Attacking

Your bot must decide which Server to attack. To attack, your server needs to send qubits to the victim machine via the network. Routing is automatic but your server has to make an investment in the connection by giving each router on the path custody of one qubit from your server.

Let's say there are \\(k\\) routers on a certain connection as illusrtrated below. When the connection is broken, some of the qubits, say \\(k \%\\), which were held by the routers are returned to your server and the rest to the victim. Your server must specify this \\(k\\) when the connection is broken.

Retaliation

Your server(\\(P_{me}\\)) is the victim of an attack from EvilServer(\\(P_{es}\\)) along a path that was \\(N\\) routers long. This means EvilServer invested \\(N\\) qubits and is now sending your server maligned qubits.

Your server cannot defend itself but it sure can retaliate! When your server decides to retaliate, it will have to make a similar investment in the routers, $$investment\ =\ \left\lfloor \frac{P_{me}}{P_{es}} \right\rfloor \times N$$ The same amount of qubits will be returned to the EvilServer.
